Compra de referencias

Rama features/compra

Te ha contratado la empresa We♥️Developers para que desarrolles una aplicación de comercio electrónico y, más específicamente, el backend.

En este reto te piden que implementes la operativa de modificación de tu catálogo.

La operativa es la siguiente:

  1. El sistema mostrará un listado de productos.
  2. El sistema pedirá que introduzca una referencia del producto del cual deseas comprar algunas unidades.
  3. El usuario introducirá una referencia0.
  4. El sistema pedirá cuántas unidades de la referencia introducida quiere comprar.
  5. El usuario introducirá un número1.
  6. El sistema actualizará la fecha de modificación (recuerda el formato)2.

0 NO SE VALIDARÁ la introducción de la referencia.

1 El número de unidades resultante de la compra NO deberá ser > 127. En caso de que el número de unidades introducido por el usuario supere las 127 unidades (de la referencia indicada), se pondrá a 127 el número de unidades de esa referencia.

2 El sistema pondrá un día y hora diferentes con respecto a la fecha de alta del producto. En caso de que la referencia a modificar cuente con una fecha de modificación, la nueva deberá ser diferente a la existente.

Se pide:

  1. Implementar la lógica indicada para mostrar la compra de referencias.
  2. Aplicar el flujo git propuesto por la empresa.
