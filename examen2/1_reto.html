Rama features/alta

Te ha contratado la empresa We♥️Developers para que desarrolles una aplicación de comercio electrónico y, más específicamente, el backend.

En este reto te piden que des de alta una lista de productos. Cada producto tiene las siguientes propiedades:

  • ID 0: clase UUID.
  • Referencia 1: una cadena de texto de 7 caracteres. Invéntate el formato. No te preocupes, el usuario será coherente con el formato.
  • Núm. de unidades 1: byte. No puede almacenar valores mayores de 127.
  • Creado el 2: fecha. Usa el formato "AAAAMMDDTHH:mm" (ej.: 20250112T30:30 para indicar el 12 de enero de 2025 a las 12 y media.).
  • Modificado el 3: fecha. Usa el formato "AAAAMMDDTHH:mm" (ej.: 20250112T45:30 para indicar el 12 de enero de 2025 a la 1 menos cuarto.).

0 Lo genera el sistema (no se pide al usuario). Se genera según se puede ver aquí .

1 Se pide al usuario.

2 El sistema la establece según el momento de su creación.

3 En esta alta, esta fecha se mantendrá en null.

El usuario puede introducir un número indeterminado de productos.

El usuario solo puede introducir un producto cada vez.

Se pide:

  1. Implementar la lógica indicada de alta de productos.
  2. Aplicar el flujo git propuesto por la empresa.
