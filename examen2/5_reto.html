Colaboración en proyectos

El flujo GIT aplicado a los otros retos NO aplica a este.

Te ha contratado la empresa We♥️Developers para que desarrolles una aplicación de comercio electrónico y, más específicamente, el backend.

En este quinto reto te piden que colabores con este proyecto.

Lamentablemente, el GIT MASTER de esta empresa no se ha dado cuenta de que el proyecto tiene errores.

La operativa es la siguiente:

  1. Sigue los pasos para obtener, en tu cuenta de Github, una copia del proyecto.
  2. Clónalo en local.
  3. Cámbiate a la rama local developer y crea una subrama con tu nombre.
  4. Realiza cuantos cambios consideres a su código para que funcione y cumpla las buenas prácticas.
  5. Con los cambios que has hecho, solicita una PR para que el GIT MASTER la apruebe.

No esperes a que el GIT MASTER apruebe la PR durante la prueba: tu misión es completar esta última.

Es posible que, durante la prueba, el GIT MASTER te haga comentarios a tus cambios.
